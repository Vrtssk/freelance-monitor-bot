# Web board (все объявления)

Локальный сайт с красивым отображением **всех** собранных объявлений со всех
бирж. Открывается кнопкой `🌐 Все объявления (сайт)` в главном меню бота
(URL-кнопка, ведёт на `WEB_BASE_URL`).

> **Дизайн:** тёмный дашборд по `docs/site-design.md` (редизайн 2026).
> Бывшая cream/monospace-тема из `DESIGN.md` — архив.

## Где живёт

Рендерится внутри FastAPI-сервиса (`api`), не отдельным фронтом. Стек —
**FastAPI + Jinja2** (серверный рендеринг), **HTMX** (обновление карточек/
фильтров без перезагрузки), **Alpine.js** (локальное состояние фильтра) и
**Tailwind CSS** (Play CDN, без сборки). Визуальный стиль — тёмная тема
(`#0d1117`), сайдбар, шапка, hero-бэннер, сетка статистики со sparkline,
4-колоночная лента карточек (см. `docs/site-design.md`).

- `GET /` (`api/main.py`) — все объявления, newest first (`render_jobs_page`).
- `GET /top` — топ-10 самых релевантных заказов (`render_top_page`).
- `GET /?partial=1[&src=<key>]` и `GET /top?partial=1[&src=<key>]` — только
  фрагмент сетки карточек (для HTMX-свопов); `src` фильтрует по бирже.
- `templates/` — Jinja-шаблоны: `base.html` (сайдбар/шапка/футер с CDN),
  `board.html` и `top.html` (страницы), `partials/grid.html` (карточки,
  общий для полных страниц и HTMX-частичных ответов).
- `api/board.py` — готовит контекст и рендерит шаблоны; `render_grid_partial`
  возвращает только сетку.
- Данные:
  - `/` → `db.repository.get_all_posts` (все `seen_posts`, лимит 300).
    Вакансии показываются с бейджем «вакансия» (красный).
  - `/top` → `db.repository.get_relevant_posts` (только не-вакансии),
    ранжирование `utils.relevance.compute_relevance` в момент запроса
    (отклики, свежесть, сложность, цена), берутся первые 10. На карточке —
    круговой прогресс-индикатор (SVG ring) с процентом релевантности,
    цвет зависит от score (зелёный ≥80%, синий 50–80%, оранжевый <50%).

## Навигация

Слева — сайдбар 64–72px с иконками (Все/Топ/Стат/GitHub). Вверху — шапка:
логотип, навигация, кнопка «обновить» и индикатор live (пульсирует, «N
сек назад»). На <880px сайдбар схлопывается в нижнюю tab-bar.

## Что умеет страница

- **Hero-бэннер** с градиентом и 4 мини-карточками преимуществ бота.
- **Сетка статистики** (6 карточек): метрики + sparkline (inline SVG).
- **Фильтр по источнику** (теги-переключатели, цвет источника): клик
  делает HTMX-запрос `/?partial=1&src=<key>` и подменяет `#grid`
  без перезагрузки; активная кнопка через Alpine.js.
- **Карточки**: логотип/чип источника, время, заголовок, краткое описание,
  теги тем (цвет по теме), цена (зелёным, tabular-nums), отклики, дата,
  ring-score (для `/top`), кнопки «Открыть» и «Источник».
- **Автообновление** сетки раз в 90 секунд (Alpine `setInterval` →
  `htmx.ajax`), с сохранением выбранного фильтра.
- **Адаптивная сетка** `auto-fill, minmax(300px, 1fr)` (4/3/2/1 колонки).

## Доступ

Сайт отдаётся API-контейнером на порту `8000` (проброшен на хост). Базовый
URL настраивается в `.env` через `WEB_BASE_URL` (по умолчанию
`http://localhost:8000`). Если открывать с другого устройства в сети —
укажи там LAN-IP хоста, например `http://192.168.1.50:8000`.

### Важно: кнопка в боте — НЕ URL-кнопка

Telegram **отвергает** `localhost` и приватные IP в inline-кнопках с `url`
(ошибка "Wrong HTTP URL"), из-за чего всё меню перестаёт отправляться. Поэтому
в главном меню стоит обычная callback-кнопка `🌐 Все объявления (сайт)`
(`menu:board`), которая при нажатии присылает **кликабельную текстовую ссылку**
на доску. Обычные ссылки в тексте сообщения Telegram не валидирует, так что
`localhost` работает. Чтобы открыть с телефона, пропиши в `.env` `WEB_BASE_URL`
с LAN-IP хоста.

# Web board (все объявления)

Локальный сайт с красивым отображением **всех** собранных объявлений со всех
бирж. Открывается кнопкой `🌐 Все объявления (сайт)` в главном меню бота
(URL-кнопка, ведёт на `WEB_BASE_URL`).

## Где живёт

Рендерится внутри FastAPI-сервиса (`api`), а не отдельным фронтом. Стек —
**FastAPI + Jinja2** (серверный рендеринг), **HTMX** (обновление карточек/
фильтров без перезагрузки), **Alpine.js** (локальное состояние фильтра) и
**Tailwind CSS** (через Play CDN, без сборки). Визуальный стиль задан в
`DESIGN.md` — моноширинный шрифт (Berkeley Mono → JetBrains Mono), тёплый
кремовый фон `#fdfcfc`, ASCII-маркеры вместо иконок, плоские карточки без теней.

- `GET /` (`api/main.py`) — все объявления, newest first (`render_jobs_page`).
- `GET /top` — топ-10 самых релевантных заказов (`render_top_page`).
- `GET /?partial=1[&src=<key>]` и `GET /top?partial=1[&src=<key>]` — только
  фрагмент сетки карточек (для HTMX-свопов); `src` фильтрует по бирже.
- `templates/` — Jinja-шаблоны: `base.html` (навбар/футер/хед с CDN-скриптами),
  `board.html` и `top.html` (страницы), `partials/grid.html` (карточки, общий
  для полных страниц и HTMX-частичных ответов).
- `api/board.py` — готовит контекст и рендерит шаблоны; `render_grid_partial`
  возвращает только сетку.
- Данные:
  - `/` → `db.repository.get_all_posts` (все `seen_posts`, лимит 300).
    Вакансии показываются с бейджем «вакансия».
  - `/top` → `db.repository.get_relevant_posts` (только не-вакансии),
    ранжирование `utils.relevance.compute_relevance` в момент запроса
    (отклики, свежесть, сложность, цена), берутся первые 10. На карточке —
    бейдж с процентом релевантности `[87%]`.

## Навигация

Вверху обеих страниц навбар с двумя ссылками: `Все` (`/`) и
`Топ-10 релевантных` (`/top`). Активная страница подсвечена.

## Что умеет страница

- Карточки: источник (эмодзи + название), заголовок (ссылка в новой вкладке),
  бюджет, чипы тем, краткое описание, дата.
- Фильтр по источнику (кнопки сверху): клик делает HTMX-запрос
  `/?partial=1&src=<key>` и подменяет `#grid` без перезагрузки; активная
  кнопка подсвечивается через Alpine.js.
- Автообновление сетки раз в 90 секунд (Alpine `setInterval` → `htmx.ajax`),
  с сохранением выбранного фильтра.
- Адаптивная сетка (`auto-fill, minmax(330px, 1fr)`), светлая «manpage»-тема.

## Доступ

Сайт отдаётся API-контейнером на порту `8000` (проброшен на хост). Базовый
URL настраивается в `.env` через `WEB_BASE_URL` (по умолчанию
`http://localhost:8000`). Если открывать с другого устройства в сети — укажи
там LAN-IP хоста, например `http://192.168.1.50:8000`.

### Важно: кнопка в боте — НЕ URL-кнопка

Telegram **отвергает** `localhost` и приватные IP в inline-кнопках с `url`
(ошибка "Wrong HTTP URL"), из-за чего всё меню перестаёт отправляться. Поэтому
в главном меню стоит обычная callback-кнопка `🌐 Все объявления (сайт)`
(`menu:board`), которая при нажатии присылает **кликабельную текстовую ссылку**
на доску. Обычные ссылки в тексте сообщения Telegram не валидирует, так что
`localhost` работает. Чтобы открыть с телефона, пропиши в `.env` `WEB_BASE_URL`
с LAN-IP хоста.

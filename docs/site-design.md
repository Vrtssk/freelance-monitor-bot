# Описание дизайна сайта (web board)

> Спецификация визуального стиля веб-доски (`/` и `/top`). Редизайн 2026 —
> тёмный дашборд по референсу `docs/reference.png`.

## Что это за сайт

Локальная веб-доска всех собранных фриланс-объявлений бота. Две страницы:

- `GET /` — все объявления (newest first), с вакансиями.
- `GET /top` — топ-10 релевантных (без вакансий), с бейджем релевантности.

## Стиль (редизайн по референсу)

Современный **тёмный дашборд**, не кремовая manpage-тема. Цель — красивый,
информативный, «дашборд-продукт», а не «README».

### Типографика

- **UI-шрифт:** системный sans (`Inter` по возможности через Fontsource/CDN,
  fallback `ui-sans-serif, system-ui, -apple-system, Segoe UI, Roboto, sans-serif`).
- **Цифры/числа:** `tabular-nums`, font-feature-settings для ровных колонок.
- Числа в статах и бюджетах — моноширинные цифры.
- Раньше был 100% monospace (Berkeley Mono) — от этого ушли: моноширинный
  шрифт только для чисел/бейджей score, для остального — чистый sans-serif.

### Радиусы

Скруглённые углы у всех элементов: `--r-card: 12px`, `--r-pill: 999px`,
`--r-btn: 8px`, `--r-input: 8px`.

### Тени/глубина

Без жёстких теней; глубина через тонкие границы `border` и лёгкий
`background`-контраст. Hover-карточка — чуть приподнятый border + тонкая
`box-shadow: 0 4px 20px -10px rgba(0,0,0,.6)`.

## Раскладка

### Каркас (base.html)

```
┌────────────────────────────────────────────────────────────┐
│ [sidebar 64px] │   [header 56px]                            │
│                │   ┌──────────────────────────────────────┐ │
│  [Все]         │   │ main  max 1280px, padding 24px        │ │
│  [Топ]         │   │                                       │ │
│  [Стат]        │   │  {page hero}                          │ │
│  [Github] ↓    │   │  {content}                             │ │
│                │   │                                       │ │
└────────────────┴───┴──────────────────────────────────────┴─┘
```

- **Сайдбар** (фикс. слева, 64–72px, тёмно-панельный): вертикальные кнопки
  с иконками (SVG inline). На мобильных убирается → нижняя tab-bar.
- **Шапка** справа от сайдбара: логотип/название слева, навигация по
  центру (`Все`, `Топ`, `Статистика`), справа — кнопка обновить (`🔄`) и
  индикатор `live` (пульсирующая зелёная точка + «обновлено N сек назад»).

### Страница «Все объявления» (`board.html`)

1. **Hero-бэннер** — большой блок с градиентом (фиолетово-синее свечение,
   radial-gradient на фоне `panel-2`), слева крупный белый заголовок
   («Все объявления») и подзаголовок-описание бота; справа или внизу —
   4 мини-карточки преимуществ (мониторит 4 биржи, гибридная фильтрация,
   LLM-классификация, автообновление). Мини-карточки: иконка + короткое
   название + одно предложение.
2. **Сетка статистики** (6 карточек по 2 ряда × 3): метрики — «Всего в
   базе», «Прислано уведомлений», «Вакансий отсеяно», «Средняя цена»,
   «Источников включено», «Свежих за 24ч». В каждой: название, крупное
   число, процент изменения (зелёный «+N%» / красный «−N%»), и
   минималистичный **sparkline** (inline SVG из 6–8 точек, без осей).
   Цвет линии совпадает с цветом карточки.
3. **Фильтры по источнику** — горизонтальная линия тегов-переключателей;
   каждый подсвечивается своим цветом источника; `Все` — нейтральный.
   Активный фильтр — залитый, неактивный — outlined. Слева маленький
   счётчик «N объявлений».
4. **Лента** — сетка карточек 4 колонки на wide, 3 на desktop, 2 на
   tablet, 1 на mobile (`auto-fill, minmax(300px, 1fr)`).

### Страница «Топ релевантных» (`top.html`)

- Тот же hero, но заголовок «🔥 Топ-10 релевантных» и подпись с формулой.
- Сетка статистики та же, но убрать (или упростить) — оставить только
  hero + лента топа (не перегружать). Решение: короткий hero + лента.
- Бейдж релевантности — **круговой прогресс-индикатор** (SVG ring) в
  каждой карточке с процентом по центру; цвет ring зависит от score:
  - >=0.8 — green
  - 0.5–0.8 — blue
  - <0.5 — orange

### Карточка объявления (`partials/grid.html`)

```
┌─────────────────────────────────────┐
│ [источник эмодзи/чип]      [ring %] │ ← если есть score
│                                     │
│ Заголовок объявления (16px semi)     │
│ Краткое описание (2–3 строки clamp) │
│                                     │
│ [тег темы] [тег темы]                │
│                                     │
│ ₽ 80 000            · 3 отклика      │
│ 12.07 14:32  · fl.ru                 │
│                                     │
│ [Открыть →]  [Источник ↗]            │
└─────────────────────────────────────┘
```

- `panel` фон, `border` 1px, `--r-card`.
- Вакансия — красная полоска/бейдж «Вакансия» слева + приглушённый текст.
- Цена — зелёным, крупнее, `tabular-nums`. Если не указана — muted «—».
- Кнопки: «Открыть →» (primary blue outline) и «Источник ↗» (ghost).
  Обе открывают `r.url` в новой вкладке (кнопки дублируют для удобства).
- Hover: border-light + лёгкий shadow + capitalize title.
- Бейджи: `notified` → маленький синий чип «отправлено».

### Сайдбар-иконки

- `Все` — сетка/список (grid icon)
- `Топ` — пламя (flame)
- `Стат` — график (bar chart)
- `GitHub` — octocat-style (внешняя ссылка на репо, по желанию)

inline SVG, 20px, stroke `muted`, активный — `blue`.

## Технические требования

- **Стек:** FastAPI + Jinja2 + HTMX (partial swaps) + Alpine.js (локальное
  состояние фильтра) + Tailwind CSS (Play CDN, без сборки). Без сборки —
  CDN-only, чтобы запускать «как раньше» (`docker compose up`).
- **Адаптив:** sidebar → нижняя tab-bar на <880px; hero/сетки адаптивны;
  font-size уменьшается на mobile.
- **Тёмная тема** — основная (light не делаем сейчас).
- **Анимации:** лёгкие (pulse на live-dot, fade-in для новых карточек,
  smooth border переходы). `prefers-reduced-motion` уважаем.
- **Автообновление** сетки каждые 90 сек (как сейчас), с сохранением
  выбранного фильтра через Alpine state.
- **Спарклайны** — inline SVG, генерируются в шаблоне по данным из БД
  (последние N значений по часам/дням). Если данных мало — рисуем
  плоскую линию/заглушку.
- **Доступ:** таблицы БД; статы реализуем поверх `db.repository.count_stats`
  + лёгкая агрегация по часам для спарклайнов (см. ниже).

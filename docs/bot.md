# Bot — архитектура и визуал

Этот документ описывает устройство Telegram-бота и текущее состояние
визуального слоя (заглушки, без реального парсинга).

## Стек

- Python 3.11+ (локально проверено на 3.14)
- `aiogram` 3.x — Telegram Bot API
- `pydantic-settings` — конфигурация из `.env`
- Режим разметки сообщений: **HTML**

## Структура

```
config/
  settings.py     — загрузка настроек из .env (BOT_TOKEN, LLM_*, SCRAPE_INTERVAL)
  topics.py       — определения тем, ключевые слова, источники, DEMO_POSTS
bot/
  main.py         — точка входа: создание Bot/Dispatcher, запуск long-polling
  state.py        — in-memory состояние пользователя (выбранные темы, пауза демо)
  keyboards.py    — все inline-клавиатуры (меню, темы, демо, настройки)
  handlers/
    start.py      — /start, главное меню, /help
    topics.py      — выбор тем, переключение toggle
    demo.py        — пример объявления (format_post + навигация)
    settings.py    — пауза/возобновление мониторинга, сброс тем
    stats.py       — статистика (тестовые данные)
run.py            — python run.py запускает бота
```

## Экраны (inline-навигация)

1. **Главное меню** (`/start`) — 4 действия + помощь.
2. **Мои темы** — список тем с галочками; тап переключает тему на месте.
3. **Демо-объявление** — реалистичный пример поста по каждой теме,
   кнопка «➡️ Следующее» листает примеры.
4. **Настройки** — статус мониторинга, пауза/возобновление, сброс тем.
5. **Статистика** — тестовые цифры по источникам и темам.

Все сообщения редактируются на месте (`edit_text`), без спама новыми
сообщениями при навигации по меню.

## Состояние пользователя

Пока используется `bot/state.py` (in-memory `dict`). При перезапуске бота
выбор тем сбрасывается. В дальнейшем заменится на SQLite (см. AGENTS.md).

## Запуск (локально, без Docker)

```bash
python3.14 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python run.py
```

Токен бота лежит в `.env` (не коммитится). Подключение проверяется через
`bot.get_me()` при старте.
